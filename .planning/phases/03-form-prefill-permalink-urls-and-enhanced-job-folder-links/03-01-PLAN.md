<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Server `/generate/prefill` | User-supplied `folderPath` is parsed, then resolved to a real directory; must remain inside the jobs root |
| Browser → Server `/generate/` (generation) | User-supplied `permalinkUrl` is validated before being written to disk |
| Server → Filesystem `permalink.txt` | The server writes a single text file to the job folder; the file content is validated |
| Server → Filesystem `job-description.txt` / `other-input.txt` / `full-jd.txt` | Read-only file reads; bounded by the jobs-root allow-list |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | `POST /generate/prefill { folderPath: '../etc' }` | high | mitigate | The prefill route uses the same `safeRealpath` + `pathIsInsideDir(realJobDir, ensureJobsRootRealpath)` pattern as `GET /generate/diffResume`. Path-traversal payloads return 400 with `folderPath escapes jobs root`. A test asserts this behaviour (mirrors `routes/generate.test.ts:600-610`). |
| T-03-02 | Information Disclosure | `/generate/prefill` returns other-input.txt content (company, role, link) to any caller | low | accept | The data is the user's own job application data on their local single-user tool. The endpoint is read-only and the same allow-list applies as for the diff endpoint, which is the established pattern (DIFF-03 in Phase 1). |
| T-03-03 | Tampering | `permalink.txt` written with attacker-controlled `permalinkUrl` body field | medium | mitigate | The `validatePermalinkUrl` helper requires the URL to start with `http://` or `https://`, contain `#job=`, and end with the correct slug. On mismatch, log a warning and skip the write — generation is not failed. The file is single-line text; no executable content. |
| T-03-04 | Repudiation | User denies opening a permalink they did open | low | accept | Out of scope. The server has no audit log requirement for permalink loads. |
| T-03-05 | Denial of Service | Pre-fill endpoint hit with many concurrent requests reading the same files | low | accept | Single-user local tool. Each read is <50 KB; the filesystem handles it trivially. No additional rate limiting required. |
| T-03-06 | Elevation of Privilege | `resolveJobFolder` returns a path outside the jobs root | low | mitigate | The returned path is then run through `safeRealpath` + `pathIsInsideDir` in the route handler; any escape returns 400. The pre-route helper is a convenience, not a security boundary. |
| T-03-07 | Spoofing | URL hash manipulated to claim a different slug | low | accept | The hash is parsed and the slug is fed to `/generate/prefill`; the server resolves the slug to a real directory inside the jobs root. A bogus slug returns 404. |
| T-03-08 | Tampering | npm/pip/cargo installs | high | mitigate | Phase 1/2/3 does not introduce new dependencies. `package.json` is unchanged. The `pre_commit_code_health_safeguard` is the existing gate; no [ASSUMED] packages. |
