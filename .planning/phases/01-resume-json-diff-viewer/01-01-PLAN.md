    2. `resumesAreEqual(a: unknown, b: unknown): boolean` — copy the body verbatim from `services/fixSuggestionsService.ts` lines 74-76 (`return canonicalize(a) === canonicalize(b);`). Make it an exported named function.
    3. `unifiedDiffText(a: string, b: string, labelA: string, labelB: string): string` — new function. Implementation (do NOT call `require('node:diff')` or any external library):
       a. If `a === b` (byte-identical strings), return `''`.
       b. Otherwise split `a` on `\n` and `b` on `\n` (preserve trailing empty lines by not trimming; use `split('\n')` so `\n` at the end yields an extra empty string — that is fine and matches `diff -u` output).
       c. Compute the common prefix length: walk forward from index 0 while `aLines[i] === bLines[i]`. Compute the common suffix length: walk backward from the end while `aLines[aLen - 1 - k] === bLines[bLen - 1 - k]`, where `k < aLen - prefix` and `k < bLen - prefix` (so the middle regions can be empty if all changed).
       d. The "changed middle" in `a` is `aLines.slice(prefix, aLen - suffix)`. The "changed middle" in `b` is `bLines.slice(prefix, bLen - suffix)`.
       e. Format the output exactly as:
          ```
          --- {labelA}
          +++ {labelB}
          @@ -{aStart},{aCount} +{bStart},{bCount} @@
           {prefix lines, each prefixed with ' '}
          -{removed line}
          ...
          +{added line}
          ...
          ```
          Where `aStart = prefix + 1` (1-indexed), `aCount = max(1, aLen - prefix - suffix)` (the `+1` rule from `diff -u` so an empty hunk still shows count 1), same for `bStart`/`bCount`. When `aCount` is 0, emit `0` not `1` (the `-0,0` form). When `bCount` is 0, emit `+0,0`. Use the `@@ -A,B +C,D @@` format with literal spaces around the dash and plus.
          - If `prefix > 0`, include all `prefix` lines each prefixed with a single space.
          - Then all `removed` lines each prefixed with `-`.
          - Then all `added` lines each prefixed with `+`.
          - If `suffix > 0`, include all `suffix` lines each prefixed with a single space, after the hunk.
       f. End the output with a single trailing newline so the modal's `<pre>` renders cleanly.
    4. `summariseJsonDiff(a: unknown, b: unknown): { changedPaths: string[]; addedKeys: string[]; removedKeys: string[] }` — new function. Implementation:
       a. Internal recursive walker `walk(aVal: unknown, bVal: unknown, prefix: string, acc: { changedPaths: string[]; addedKeys: string[]; removedKeys: string[] }): void`:
          - If `aVal` and `bVal` are both plain objects (typeof === 'object', not null, not array):
            - For every key in `aVal` not in `bVal` → push `prefix + '.' + key` into `acc.removedKeys` (do NOT recurse; just record the path).
            - For every key in `bVal` not in `aVal` → push `prefix + '.' + key` into `acc.addedKeys`.
            - For every key in both → recurse with `prefix + '.' + key`.
            - Use `Object.keys` (do not enumerate symbols).
            - Skip the empty-prefix case at top level: when `prefix === ''`, omit the leading dot. (Implement with a helper that joins a base and a key, only adding the dot when base is non-empty.)
          - If both are arrays of equal length: iterate index-by-index and recurse for each element with prefix `prefix + '.' + i`. For length mismatch or non-equal arrays, mark `prefix` as a changed path.
          - For scalar mismatch (or type mismatch: object vs scalar, array vs scalar) → push `prefix` into `acc.changedPaths`.
       b. Public function returns `{ changedPaths, addedKeys, removedKeys }` with the three arrays.

    File-level rules (per CONVENTIONS.md):
    - No `default export`. All four symbols are named exports.
    - No new runtime dependencies without evaluation. Well-tested libraries are preferred for complex problems.
    - Add a one-line top-of-file `//` comment naming the source plan: `// services/diffUtil.ts — shared diff helpers for the in-app resume diff viewer (RESUME_DIFF_VIEWER_PLAN.md Step 1).`
    - Keep the file under 120 lines; each function under 50 lines (the recursion in `summariseJsonDiff` should split out a private `walk` helper).
