  <name>Task 1: Rewrite unifiedDiffText as LCS-based multi-hunk algorithm (G-01-2 RC-1)</name>
  <files>services/diffUtil.ts, services/diffUtil.test.ts</files>
  <behavior>
    - Input: two multi-line strings a and b, labelA, labelB
    - Output: unified diff string with separate @@ hunks for each changed region
    - Case: identical inputs → return ''
    - Case: single change in middle → one hunk with ~3 context lines before/after
    - Case: three disjoint changes → three separate @@ hunks
    - Case: added lines → output with + prefix, removed lines → output with - prefix
    - Case: context lines → output with space prefix
    - Performance: handles ~30 KB input in under 50 ms
  </behavior>
  <action>
Rewrite `unifiedDiffText` in services/diffUtil.ts using an LCS-based multi-hunk algorithm:

1. Compute line-by-line LCS between before[] and after[] arrays to find all changed line ranges.

2. Walk the LCS output to identify maximal contiguous changed regions (runs of insert/delete operations).

3. For each changed region:
   a. Compute the start/end indices in before and after
   b. Add up to 3 lines of context before the changed region
   c. Emit the hunk header: `@@ -<start>,<count> +<start>,<count> @@`
   d. Emit context lines (space-prefixed), removed lines (-), and added lines (+)

4. Output format per hunk:
   - `--- labelA` / `+++ labelB` headers (once at the top of the full output, NOT per hunk)
   - Per hunk: `@@ -<beforeStart>,<beforeLen> +<afterStart>,<afterLen> @@`
   - Context lines prefixed with ' '
   - Removed lines prefixed with '-'
   - Added lines prefixed with '+'

5. Preserve the existing performance requirement: handles ~30 KB in under 50 ms.

6. **Do NOT change the function signature** — it must remain `unifiedDiffText(a: string, b: string, labelA: string, labelB: string): string`.

7. Keep `countCommonPrefix`, `countCommonSuffix`, and `LinePair` in the file — they may be referenced by tests.

After the rewrite, add a new test to `services/diffUtil.test.ts`:
```typescript
it('produces separate hunks for multiple disjoint changes', () => {
  const a = 'line0\nline1\nline2\nline3\nline4\nline5\n';
  const b = 'line0\nMODIFIED1\nline2\nline3\nMODIFIED4\nline5\n';
  const out = unifiedDiffText(a, b, 'a', 'b');
  // Should have two @@ markers (two separate hunks), not one
  const hunkCount = (out.match(/^@@ /gm) || []).length;
  expect(hunkCount).toBe(2);
});
```
